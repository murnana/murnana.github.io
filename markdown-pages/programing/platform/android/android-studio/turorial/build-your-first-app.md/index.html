初めてのアプリの作成

Android Studio をダウンロードする

| https://developer.android.com/studio/?hl=ja からダウンロードする | ほぼデフォルトのままインストール

Android Studio の初回起動

インストールが終了し、初めて Android Studio を立ち上げると、セットアップ画面になる。

前のバージョンのアップデートなどでない限りは「Do not import settings」を選ぼう

その後、Setup Wizard が立ち上がる

なんかもう早速アップデートあるよ！と言っているがひとまず無視する

何も考えたくないなら「Standard」を選ぶ。 Custom にして、必要な SDK だけ入れてもいい (SDK は後でも入れられる)

IDE のテーマを選べる。私はダークテーマが好きなので「Darcula」を選んだ

セットアップの最終チェック。SDK は Windows の場合 %LOCALAPPDATA%\Android\Sdk が デフォルトになる

スクロールすると他に何が入るのかが分かる。問題なければ「Finish」を押す

後はひたすらダウンロードが終わるのを待つ

プロジェクトの新規作成

このまま新規プロジェクトを始めてもよいが、その前に一工夫

右下の歯車マーク(Confingure)から「Settings」を選択。

その中の「System Settings」から「Statup/Shutdown」の項目を選び、「Reopen last project on startup」にチェックを入れる。

これにより、起動時に Welcome ページが開くようになる。(前回のプロジェクトを開くようにしたい場合は、このチェックを外す)

話を戻して、「Start a new Android Studio project」を押す

3.3 から Create New Project wizard が変わっている。

「Empty Activity」を選択

Name にプロジェクト名、Save location にプロジェクト名+保存位置を入れる

Finish 後、必要な依存パッケージがダウンロードされる

スタート直後は、プロジェクトを読み込み切っていない状態なので注意

Gradle プロジェクトが読み込み終わったら OK。

エミュレータ or 実機での確認

アプリケーションをビルドし、実機またはエミュレータに実行させるには、Run を押す

その後、実機又はエミュレータを選ぶ

実機の場合

| 実機に「開発者向けオプション」を表示させる | そのためには、以下の手順が必要

Android 8.0 以降:

「設定」>「バージョン情報」>「ソフトウェア情報」>「その他」>「ビルド番号」

この「ビルド番号」を 7 回タップする

その後、「開発者向けオプション」から「USB デバック」を ON にする

その状態で PC に接続すると、「Select Deployment Target」にそのデバイスが現れるので、選択して「OK」を押す

エミュレータの場合

新規作成時は「Create New Virtual Device」を押下。そうでなければデバイスのリストが並ぶので、それを選んで実行できる。

エミュレートしたいデバイスを選ぶ

デバイスの AndroidOS バージョンと ABI{.interpreted-text role="term"} を選択

「AVD Name」にエミュレータの名前、「Startup orientation」に開始時の向きを入れる
